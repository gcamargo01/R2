R2 services application program interface.

List of different kind of R2 modules implementations

{@link uy.com.r2.core.api.SimpleService} - The simplest interface to implement e service. It is a {@link uy.com.r2.core.api.Module} with one main method:
  SvcResponse call( SvcRequest)

{@link uy.com.r2.core.api.AsyncService} - The asynchronous way to implement a service module, it handles request and response in two times, with the methods:
  SvcMessage onRequest( SvcRequest)
  SvcRequest onResponse( SvcRequest)

Each service has a requests and a responses

{@link uy.com.r2.core.api.SvcRequest} - Service request message information: Service, Data, ...
{@link uy.com.r2.core.api.SvcResponse} - Response message, with a ResultCode:
  Zero - A normal response
  Positive - A kind of error condition, example: Not found
  Negative - Fatal error, some infrastructure is failing

Support methods (from {@link uy.com.r2.core.api.Module} interface)

  {@link uy.com.r2.core.api.ConfigDescriptor} getConfigDescriptors() - Sets the name, type and attributes of each parameter that controls the operation of the service.
  Map<String,Object> getStatusVars() - Get service status indicators to monitor and diagnose.
  shutdown() - Stop the service execution, so release its resources.
